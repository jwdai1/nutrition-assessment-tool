# 低栄養診断ツール（MNA-SF / GLIM）

MNA-SF スクリーニングと GLIM 基準による低栄養診断・重症度評価を行う PySide6 デスクトップアプリケーションです。

---

## 機能

- **患者管理** — 患者の登録・編集・削除・検索
- **アセスメントウィザード** — 4ステップの対話形式で評価を入力
  - ステップ1: 患者基本情報（身長・体重・BMI自動計算）
  - ステップ2: MNA-SF スクリーニング（体重減少・BMI を自動推定）
  - ステップ3: GLIM 評価（体重減少・低BMI を自動判定）
  - ステップ4: 診断結果・推奨アクションの表示
- **評価履歴** — 患者ごとの評価記録を一覧表示
- **PDF出力** — 評価レポートをA4 PDFとして保存
- **SQLite 永続化** — データはローカルに安全に保存

## インストール

### 1. リポジトリをクローン

```bash
git clone https://github.com/jwdai1/nutrition-assessment-tool.git
cd nutrition-assessment-tool
```

### 2. PySide6 をインストール

```bash
pip install PySide6
```

> macOS で複数の Python 環境がある場合は `pip3` または特定の Python を指定してください。

### 3. 起動

```bash
python main.py
```

## 使い方

### 患者の登録

1. 左パネル下の **「＋ 患者追加」** ボタンをクリック
2. 患者ID（必須）・氏名・性別・生年月日・備考を入力して **「保存」**

> 患者IDは一意である必要があります（例: `PT-0001`）

### 評価の実施

1. 左パネルの患者リストから対象患者を選択
2. 右パネル下の **「新規評価」** ボタンをクリック
3. ウィザードに沿って4ステップを入力

#### ステップ 1: 患者基本情報

| 入力項目 | 説明 |
|----------|------|
| 評価日 | カレンダーから選択（デフォルト: 今日） |
| 年齢 | 評価時の年齢（歳） |
| 身長 / 現在体重 | 必須。BMI・体重減少率を自動計算して表示 |
| 3 / 6ヶ月前体重 | 入力すると体重減少率を自動算出。不明な場合は「不明」にチェック |
| 下腿周囲長 CC | BMI が計算できない場合に MNA-SF Q-F の代替として使用 |

#### ステップ 2: MNA-SF スクリーニング

- 問A〜F の6問を選択（14点満点）
- **問B**（体重減少）と **問F**（BMI / CC）はステップ1のデータから自動推定
- 自動推定値は確認の上、臨床的判断で変更可能
- リアルタイムでスコアバーに反映

**MNA-SF判定基準:**

| スコア | 判定 |
|--------|------|
| 12〜14点 | 栄養良好（GLIM評価省略） |
| 8〜11点 | 低栄養のリスクあり |
| 0〜7点 | 低栄養の可能性 |

#### ステップ 3: GLIM 評価

MNA-SF が 12点未満の場合のみ実施します。

**表現型基準**（1項目以上で充足）:

| 基準 | 自動判定 |
|------|----------|
| 意図しない体重減少 | ステップ1の体重データより自動入力 |
| 低BMI（アジア人基準） | 70歳未満: <20 / 70歳以上: <22 で自動判定 |
| 筋肉量・筋量の低下 | コンボボックスで手動選択（なし / 軽中等度 / 高度） |

**病因基準**（1項目以上で充足）:

| 基準 | 入力方法 |
|------|----------|
| 食事摂取量の低下 / 消化吸収障害 | チェックボックス |
| 急性疾患 / 外傷による炎症 | チェックボックス |
| 慢性疾患による炎症（中等度） | チェックボックス |

**GLIM診断:** 表現型基準 **かつ** 病因基準が両方充足された場合に低栄養と診断

**重症度:**

| Stage | 基準 |
|-------|------|
| Stage 1（中等度） | 体重減少5〜10%（3M）/ 10〜20%（6M）、BMI軽度低下、筋量軽中等度低下 |
| Stage 2（高度） | 体重減少>10%（3M）/ >20%（6M）、BMI高度低下、筋量高度低下 |

#### ステップ 4: 結果確認・保存

- 診断結果バナー・患者サマリ・GLIM評価内訳・推奨アクションを表示
- **「保存」** ボタンで評価をデータベースに記録

### 評価履歴の確認

- 患者を選択すると右パネルに評価履歴が一覧表示
- 行を**ダブルクリック**で詳細サマリダイアログを表示

### PDF 出力

1. 履歴テーブルで対象の評価行を選択
2. **「PDF出力」** ボタンをクリック
3. 保存先を指定してA4 PDFとして保存

## データ保存場所

評価データは以下のパスに SQLite データベースとして保存されます。

```
~/.nutrition_tool/data.db
```

バックアップする場合はこのファイルをコピーしてください。

## ファイル構成

```
nutrition_tool/

├── wizard.py        # アセスメントウィザード（4ステップ QDialog）
└── pdf_export.py    # PDF出力（QPrinter + QPainter）
```
